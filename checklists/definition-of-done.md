# Definition of Done

A task is done only when:

- The relevant spec exists or was updated.
- Acceptance criteria are satisfied.
- Tests were added or updated.
- Tests pass locally or the failure is documented.
- Lint and type checks pass where applicable.
- Documentation was updated.
- ADR was added or updated if needed.
- Security implications were reviewed.
- Generated AI claims are evidence-grounded.
- Limitations and confidence are preserved.
- No unrelated changes were included.
